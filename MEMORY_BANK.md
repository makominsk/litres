# MEMORY BANK — Litres Project

> Контекст проекта для Claude Code. Обновляется по ходу разработки.

---

## Что это за проект

Голосовой помощник по истории Древнего мира для учеников **5 класса**. Ученик отвечает голосом на вопросы из учебника, AI поясняет ответы, показывает иллюстрации и карты событий. В конце — тест по датам.

**Аудитория:** дети 10-11 лет. Тон — дружелюбный, поддерживающий, никогда не критикует.

## Ключевые архитектурные решения

### Голосовой ввод
- Web Speech API (Chrome/Edge) + текстовый fallback для остальных браузеров
- Автоопределение поддержки

### ElevenLabs (экономия)
- Вопросы и вступления — pre-generated, хранятся в `/public/audio/`
- Пояснения к ответам — на лету, кэшируются в `audio_cache` (Supabase Storage)

### Прогресс без авторизации
- Ученик вводит никнейм → UUID в Supabase + LocalStorage
- Можно продолжить с другого устройства

### Gemini промпт (важно!)
```
Ты дружелюбный учитель истории. Ученик 5 класса отвечает на вопрос.
Его речь может содержать ошибки в произношении терминов.
Интерпретируй ответ максимально лояльно.
Никогда не говори "неправильно" — только "ты на верном пути, давай уточним..."
Пояснение: 5-10 предложений + интересный факт + сравнение с современностью.
Только образовательный контент. Простой язык. Без насилия/политики.
```

## Supabase таблицы

- `students` — никнейм → UUID
- `answers` — ответы на вопросы параграфа (правильно/нет, уровень подсказки)
- `quiz_results` — результаты тестов по § и разделам
- `audio_cache` — кэш TTS (text_hash → audio_url)

## Структура страниц

```
/                        → карта мира, выбор раздела
/section/[id]            → список §§ + прогресс
/paragraph/[id]          → вопросы + карта + иллюстрации
/paragraph/[id]/quiz     → тест по датам
/paragraph/[id]/review   → повторение ошибок
/section/[id]/quiz       → обобщающий тест
/section/[id]/summary    → итоги раздела
```

## API routes

```
POST /api/evaluate    → Gemini анализирует ответ ученика
POST /api/tts         → ElevenLabs + кэш
POST /api/hint        → подсказка уровня 1/2/3
POST /api/student     → создать/найти ученика
GET  /api/progress    → прогресс ученика
POST /api/progress    → сохранить прогресс
```
